
# release notes after Review 1

- добавлена поддержка аргумента `--tickers-file`;
- Вместо `--server-addr` и `--udp-port` решил обойтись одним `--target-quote-server`, который сразу парсится как сокет и добавлена ошибка ;
- клонирование tcp-стрима обернуто выполнено через match. Если клонировать поток не получится, то ошибка залогируется , а серивис продолжит работу. В TCP-поток будет отправлена ошибка.;
- Обработана очистка и регистрация клиента (match вместо unwrap);
- добавлениа fn `get_cur_timestamp()` для получения меток времени;

PS: unwrap_or для дефолтных портов решил оставить , т.к. кажется в этом случае не будет профита от match или map_err.

- [x] Клиент не поддерживает --tickers-file
По заданию клиент должен принимать аргумент --tickers-file и читать список тикеров из файла. Сейчас тикеры задаются только позиционным аргументом (например, 127.0.0.1:8001 AAPL,MSFT). - [x] Использование unwrap() в продакшн-коде
    В бинарниках есть вызовы unwrap(), из-за которых процесс может падать при неожиданных условиях:
bins/quote_server/main.rs:258:
    - tcp_stream.try_clone().unwrap()
при ошибке клонирования сокета сервер падает вместо аккуратного ответа клиенту и логирования.
    - bins/quote_server/main.rs:277, 321: clients.lock().unwrap()
при отравленной мьютексе (panic в другом потоке, державшем lock) весь сервер падает.
    Эти места при редких сбоях (или при некорректном вводе) могут привести к падению. Нужно заменить их на обработку через Result/? или match с логированием и корректным выходом (без unwrap()/expect() в основном пути).
