
# Review 1
## Что обязательно нужно исправить
- [x] Клиент не поддерживает --tickers-file
По заданию клиент должен принимать аргумент --tickers-file и читать список тикеров из файла. Сейчас тикеры задаются только позиционным аргументом (например, 127.0.0.1:8001 AAPL,MSFT). Без --tickers-file не выполняется требование «файл/аргумент с тикерами» и усложняется использование больших списков тикеров (нельзя просто передать файл).
- [ ] Использование unwrap() в продакшн-коде
    В бинарниках есть вызовы unwrap(), из-за которых процесс может падать при неожиданных условиях:
bins/quote_server/main.rs:258:
    - tcp_stream.try_clone().unwrap()
при ошибке клонирования сокета сервер падает вместо аккуратного ответа клиенту и логирования.
    - bins/quote_server/main.rs:277, 321: clients.lock().unwrap()
при отравленной мьютексе (panic в другом потоке, державшем lock) весь сервер падает.
    Эти места при редких сбоях (или при некорректном вводе) могут привести к падению. Нужно заменить их на обработку через Result/? или match с логированием и корректным выходом (без unwrap()/expect() в основном пути).
## Что можно улучшить (рекомендации)
- [ ]  CLI-аргументы клиента: добавить явную поддержку --server-addr и --udp-port. При неверных значениях выводить короткое сообщение об использовании и выходить с ненулевым кодом, вместо возможного panic (как в пункте про unwrap() выше).
- [ ] Читаемость и дублирование: в lib.rs в QuoteBatch::new используется SystemTime::now().duration_since(UNIX_EPOCH).map(...).unwrap_or(0)
лучше вынести получение текущего времени в одну функцию/хелпер и не дублировать похожую логику с unwrap_or(0) в других местах.
